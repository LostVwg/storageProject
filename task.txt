Реализовать хранилище (в памяти) имплементирующее IStorage интерфейс

Методы интерфейса:
    1) добавить элемент в хранилище. Метод принимает в качестве аргументов ключ с типом строка и значение с типом любое, а возвращает пусто
    2) взять элемент из хранилища. Метод принимает в качестве аргумента ключ с типом строка, а возвращает значение любого типа
    3) убрать элемент из хранилища. Метод принимает в качестве аргумента ключ с типом строка, а возвращает пусто
    4) очистить хранилище. Методо ничего не принимает и ничего не возвращает
    5) акцессор позволяющий узнать количество ключей в хранилище. Не позволяет изменять значение, только получать. Возвращает число в качестве значения.

При отсутствии значения вернуть нул.

По возможность написать юнит тесты на полученный класс, покрывающие 3 случая:
    1) инициализация (при создании все поля принимают ожидаемые значения)
    2) добавление элемента (все методы позволяющие достать добавленное значение по ключу работают и размер хранилища увеличился)
    3) удаление элемента (добавление, но наоборот - возвращается нулы, размер хранилища уменьшился

Тест фреймворки: mocha / jest
По возможности оформить проектом на гитхабе
